## 关于我

我关注 **高校学术项目全生命周期治理**：从学生申报、导师审阅，到秘书处受理、专家评审、答辩与公示异议。

相信好的产品应该 **两步走**：

1. **先让人敢用** — 流程不比传统系统差，AI 为经办人省时间  
2. **再让人离不开** — 数据与图谱积累后，做传统系统做不到的治理洞察  

目前在 GitHub 上维护可交互 Demo，用于需求验证、售前演示与协作沟通。

### ARGP 能演示什么

- 多角色工作台（学生 / 导师 / 秘书处 / 专家）  
- AI 辅助写作、质量自检、**答辩前预演**  
- 秘书处退件、公示异议、结果申诉、专家回避  
- 消息待办与审计友好的流程状态机  

```bash
git clone https://github.com/bukeke111/AI-academic-review.git
cd AI-academic-review && python3 -m http.server 8765
# 打开 http://localhost:8765/login.html
```
